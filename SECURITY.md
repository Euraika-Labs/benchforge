# Security Policy

## Supported Versions

Security fixes are handled for the default branch unless a release policy is published later.

## Reporting a Vulnerability

Please do not open a public issue for suspected vulnerabilities.

Use GitHub private vulnerability reporting if available for this repository. If that is not available, contact the maintainers privately through GitHub and include:

- Affected files, versions, or configuration.
- Steps to reproduce.
- Impact and exploitability notes.
- Any suggested mitigation.

Maintainers will acknowledge valid reports as soon as practical and coordinate a fix before public disclosure.

## Secrets

Never include API keys, tokens, private keys, passwords, customer data, or credentials in issues, pull requests, logs, screenshots, or commits.
